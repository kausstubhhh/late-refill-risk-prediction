
## Steps

1. Fork the repository  
2. Create a new branch  
3. Make your changes  
4. Run the notebook end-to-end  
5. Submit a pull request  

## Guidelines

- Follow PEP8  
- Keep notebooks clean and reproducible  
- Add comments where needed  
- Update requirements.txt if dependencies change  

## Data Policy

Do NOT commit dataset files.  
Users must download CMS data separately.

## Validation

Before submitting:
- Notebook runs without errors  
- Metrics and outputs are reproducible
